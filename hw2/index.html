Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

The de Casteljau algorithm is a recursive method for evaluating Bezier curves by repeatedly performing linear interpolation between consecutive control points. At each step, given n control points and a parameter t (between 0 and 1), the algorithm generates n-1 intermediate points by linearly interpolating between each pair of adjacent points. This process continues recursively until a single point remains, which lies on the final Bezier curve at parameter t. By varying t from 0 to 1, we can trace out the entire curve.

In this part, the implementation is straigntforward as the algorithm. We just first return the point if the input is only one control point. Otherwise, we run the for loop to push back the lerp (liner interpolation) of the neighboring two points. Then, I created a test case with 6 control points (0.15, 0.25), (0.35, 0.70), (0.50, 0.85), (0.65, 0.40), (0.80, 0.65), and (0.95, 0.30). Part 6: Loop subdivision for mesh upsampling

This is probably the most complicated part of this homework. Basically, the 4-1 subdivision alrogithm aims to make the mesh smoother by adding more vertices on the edges. The process is as follows: (1) adding new vertices (computing the location of new vertices) and updating the positions of the original vertices; this is a bit tricky because it's not just adding new vertices but also updating the old ones. (2) Before adding new vertices, we need to first split the edges (creating 4 triangles from each original edge) and then doing edge flipping (all edges connecting old and new vertices) to ensure great connectivity. (3) Finally, we put the already saved, pre-computed new vertices positions back to the right place and reset all members like valence and isNew to false. The most difficult part is to calculate the new vertices but the assignment description has already provided the formula for the weights so we just implemented accordingly. Results are shown below.


As mentioned above, the subdivision is an algorithm for mesh smoothing so sharp corners get smoothed/rounded after applying that and sharp edges become curved after multiple iterations. To reduce this effort, doing pre-splitting edges newar corners can definitely help. The reason is that we just provided more vertices near these spaces so each vertex has more neighbors now. Tgerefore, the average has less smoothing effect (the u weight decreases as degree n increases: u = 3/(8n)). However, this is not a perfect solution cuz loop subdivision will eventually smoothen these areas.

When loading the default cube mesh and applying the subdivision, the cube becomes asymmetric because the mesh itself has an asymmetric topology even though it looks symmetric geometrically. Specifically, each face of the cube is split by only one diagonal edge into two triangles and they may not all go in consistent directions. Therefore, in the subdivision algorithm, vertices along these diagonal are weighted differently, resulting in asymmetry meshes. To resolve this issue, we can just do pre-splitting for each face to two diagonal edges (X-pattern on each face) such that the mesh becomes topologically symmetric.
